\section{Proof of NP-Hardness}
\label{sec:np_hard}

The Traveling Salesman Problem belongs to the class of \textbf{NP-hard} problems. This means that no algorithm is known to solve every 
instance of TSP in polynomial time, and the existence of such an algorithm would imply that $P = NP$.

\subsection{Reduction from Hamiltonian Cycle}

To demonstrate that TSP is NP-hard, we use the polynomial reduction technique from the Hamiltonian Cycle (HC) problem, which is already 
known to be NP-complete.

Let $G = (V, E)$ be an unweighted graph for which we wish to determine if a Hamiltonian cycle exists. We construct a complete weighted 
graph $G' = (V, E')$ for TSP as follows:

\begin{itemize}
    \item For each edge $(u, v) \in E$, we set the cost $w(u, v) = 1$.
    \item For each pair of nodes $(u, v) \notin E$, we set a very large cost, $w(u, v) = k$, where $k > |V|$.
\end{itemize}

Argument: If a Hamiltonian cycle exists in the original graph $G$, then in the constructed TSP graph $G'$, the minimum cost of a tour 
will be exactly $|V|$. If no Hamiltonian cycle exists in $G$, any tour in $G'$ must include at least one edge of cost $k$, so the total 
cost will be strictly greater than $|V|$.

Thus, if we had a polynomial-time algorithm to find the minimum cost in TSP, we could automatically decide the existence of a 
Hamiltonian Cycle in $G$, which demonstrates that TSP is at least as hard as any problem in the NP class.

\subsection{Consequences on Algorithms}

This classification justifies the use of three different approaches in this report:

\begin{enumerate}
    \item \textbf{Brute Force:} Exhaustive search ($O(n!)$), guaranteed to be optimal but practical only for $n \le 13$.
    \item \textbf{Held-Karp:} Dynamic programming ($O(n^2 \cdot 2^n)$), which exponentially improves execution time but remains memory-limited.
    \item \textbf{Greedy:} Polynomial-time heuristic ($O(n^2)$), which provides a solution quickly but without the guarantee of optimality 
    or even success in sparse graphs.
\end{enumerate}
